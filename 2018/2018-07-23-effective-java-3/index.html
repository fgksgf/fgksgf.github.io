Item 40

谨慎设计方法签名（方法名、返回值、形参等）

  1. 谨慎选择方法名。首要目标是选择一个易于理解的，即见名知意；第二目标是要遵守相关规范。
  2. 避免过长的参数列表。一个方法传入的参数尽量不要超过四个。
  3. 参数的类型尽量使用接口而不是类。
  4. 参数尽量使用双元素的枚举类型而不是布尔型。这使得代码的可读性更好。
  5. Don't go overboard in providing convenience methods，这句话不是很理解。
